# FinGuard AI – Fraud Detection & Risk Scoring Dashboard 🛡️

A professional, interactive ML web application for detecting financial fraud and assessing transaction risk. 

## Overview
FinGuard AI is a realistic implementation of a machine learning workflow for a FinTech scenario. The dashboard takes in transaction parameters and evaluates them using pre-trained machine learning models to determine the probability of fraud, the ultimate risk level, and the recommended action to take (Allow/Review/Block).

## Features
*   **Single Transaction Prediction:** Input data manually and receive real-time fraud probability and risk scores. Includes human-readable explanations pinpointing *why* a transaction was flagged.
*   **Bulk CSV Upload Analysis:** For batch processing. Upload a CSV of multiple transactions and receive predictions on the whole set alongside aggregated metrics.
*   **Interactive Analytics Dashboard:** Explore insights into fraud distribution by transaction amount, location mismatch, and more using beautiful Plotly charts.
*   **Model Performance Evaluation:** Detailed breakdown of model evaluation metrics (Accuracy, Precision, Recall, F1, ROC-AUC) comparing multiple models (Logistic Regression, Decision Tree, Random Forest).

## Tech Stack
*   **Backend & Data Processing:** Python, Pandas, NumPy
*   **Machine Learning:** Scikit-Learn
*   **Frontend & Data Visualization:** Streamlit, Plotly, Matplotlib
*   **Styling:** Custom native Streamlit theming focusing on a premium professional aesthetic.

## Setup & Installation

Follow these steps to set up the project locally:

1. **Navigate to the core project directory:**
   ```bash
   cd FinGuard-AI
   ```

2. **Install all required dependencies:**
   It is recommended to use a virtual environment, but either way works.
   ```bash
   pip install -r requirements.txt
   ```

3. **Generate Dataset and Train the Model:**
   The project requires a dataset and a trained model to function. Run the training script directly. This script dynamically creates a synthetic 15,000-row fraud dataset and robustly trains the machine learning pipeline.
   ```bash
   python train_model.py
   ```
   *You'll see output detailing the class distributions and the performance of various models. The best model will automatically be saved to `models/fraud_model.pkl`.*

4. **Run the Streamlit Dashboard:**
   ```bash
   streamlit run app.py
   ```
   Open the Local URL provided in your browser to view the application!

## File Structure Highlights
- `/app.py`: Streamlit frontend application.
- `/utils.py`: Simulated dataset generation tool.
- `/preprocess.py`: Contains the data preprocessing pipeline (cleaning, scaling, encoding).
- `/train_model.py`: Script coordinating automated data splitting, model training, evaluation, and exporting the top performer.
- `/data/`: Autogenerated datasets live here.
- `/models/`: Serialized top models live here.
- `/notebooks/`: For initial exploration (jupyter notebooks).
